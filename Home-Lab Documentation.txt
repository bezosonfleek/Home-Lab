Home Lab Documentation.

1. Setting up Proxmox
   -Using rufus flash proxmox onto a USB
   -Plug it into a computer and enter BIOS and continue with setup
   -Ensure it is connected to the interned (ethernet) to allow access
   -Access on web interface using ip/port given
   -Disable enterprise mode in updates and add the no-subscription option (Repositories)
   -Update via shell (apt-get update, apt-get upgrade)
   -Enable notifications
2. Setting up Windows Server
   -Download ISO file from Microsoft site
   -Upload to proxmox then create a VM, change to required needs and boot/ use a USB and make it bootable for a dedicated server
   -Select the desired version(datacenter, standard...) and set the local admin password
   -Promote server to Domain controller
   
3. Set up Active Directory, DNS & DHCP 
   -Add roles/ features, select AD DS and DNS and select the group policy features is enabled
   -Configure/promote the server as domain controller using new forest and setup desired domain name, choose and record a DSRM password
   -Ensure DNS role is included
   -Checking DNS, forward lookup zones are set after promoting to DC, create a reverse lookup zone
   -Set up DHCP server; create DHCP scope outlining the address pool, reserved addresses and gateway
   -Create an organizational unit and add users then add computer to domain - admin@domain, and login using those credentials
   -Setting up common wallpaper: create a shared file and give everyone access, go into Group policy management and create new GPO and name it, Right
    click to edit and go into the desktop option under administrative templates, Add the location to the image, Login in to client and confirm; use
    gpupdate OR gpupdate /force if it is not yet implemented
   -Setting up additional domain controller: Log into the other windows server without adding to domain, Add AD DS role, Promote to DC using credentials
    <user>@<domain>, Select the DC AD DS to replicate
   -Configure RDP (include features: Licensing & Session host for RDP to work)
   -Enabling ssh on windows server: in setting -> apps, download openssh server, got services.msc and start and set to automatic 
   -Set up ansible for automation: after configuring proxmox, in your windows download wsl (wsl --install), sudo apt install -y ansible python3-pip, 
   -Set up terraform for automation (proxmox):

Disabling root login
Count all user accounts - cut -d: -f1 /etc/passwd / wc -l /etc/passwd
List human user accounts - awk -F: '$3 >= 1000 && $3 <= 60000 { print $1 }' /etc/passwd
Check currently logged in users - who / w
Create new user - sudo adduser <username>
List members of sudo group - getent group sudo
Add to sudo group - sudo usermod -aG sudo <username>
Harden ssh - Edit /etc/ssh/sshd_config , set: 
              PermitRootLogin no
              PasswordAuthentication no (Only use SSH Keys!)     then sudo systemctl restart ssh
              If you can still login check /etc/ssh/ssh_config.d for 50-cloud-init.conf and also set PasswordAuthentication no
